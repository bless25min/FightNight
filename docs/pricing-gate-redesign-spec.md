# Fight Night 首頁入口與 LINE 解鎖改版說明

## 改版目的
首頁不再直接顯示價格方案。

新的做法是：

1. 首頁只提供 `查看活動場次 / 費用資訊` 入口
2. 使用者進入 `/offers` 頁後，先看到 `LINE 會員專屬內容`
3. 透過 LINE Login 解鎖後，才能查看完整活動場次、費用資訊、教練資訊與名額狀態

這樣做的目的不是故弄玄虛，而是讓首頁先保持品牌節奏，不過早掉進價格比較。

## 改版核心

### 1. 首頁改成單一入口
首頁不再出現：

- 價格
- 方案卡
- Pass 比較

首頁改成顯示：

- 活動資訊入口標題
- `LINE 會員專屬內容` 提示
- 解鎖內容預覽
- CTA：`查看活動場次 / 費用資訊`

### 2. /offers 頁改成會員專屬內容頁
`/offers` 不再直接表現成公開報價頁，而是先告訴使用者：

- 這裡是 LINE 會員專屬內容
- 登入後會解鎖哪些資訊
- 下方各區目前是鎖定狀態

### 3. 解鎖語氣要一致
所有相關 CTA 與區塊文案都要統一成：

- `查看活動場次 / 費用資訊`
- `快速登入查看`
- `LINE Login 後解鎖`
- `LINE 會員專屬內容`

不要再混用：

- `立即搶位`
- `選擇你的 Pass`
- `方案與報價`
- `回到方案與報價`

## 使用者流程

1. 在首頁看到 `查看活動場次 / 費用資訊`
2. 點進 `/offers`
3. 在頁首看到 `LINE 會員專屬內容`
4. 先看會解鎖哪些內容
5. 點 `快速登入查看`
6. 完成 LINE Login 後解鎖完整內容

## 首頁改版內容

### 原本
首頁 `TicketSection` 原本仍帶有：

- 價格遮罩
- Pass 卡片
- 方案導向語氣

### 現在
首頁 `TicketSection` 改成：

- 單一入口區塊
- 四個解鎖內容預覽
- 單一主按鈕
- 不再出現任何價格或方案名稱

### 首頁文案方向
首頁這段要讓人感覺：

- 我還沒看到價格
- 但我知道點進去會看到什麼
- 這是一個需要登入後查看的資訊入口

## /offers 頁改版內容

### Hero 區
Hero 不再直接說「這裡有完整方案、報價」。

Hero 現在要先建立：

- 會員專屬感
- 解鎖感
- 快速登入的行動誘因

### 預覽區
Hero 下方先補一段：

- 活動場次
- 費用資訊
- 教練資訊

讓使用者知道登入之後能得到什麼，不會只有一個空泛的登入要求。

### 下方正式內容區
下方各區改成：

- 區塊本體保留
- 內容做 blur / lock 狀態
- 上面蓋一層解鎖浮層
- 每區都有 `快速登入查看`

這樣可以明確表達：

- 內容是存在的
- 但現在還沒解鎖
- 下一步就是 LINE Login

## 文案原則

### 首頁
首頁語氣要像入口，不像購買。

適合：

- `查看活動場次 / 費用資訊`
- `LINE 會員專屬內容`
- `LINE Login 後解鎖`

不適合：

- `立即搶位`
- `立即購買`
- `選擇你的 Pass`

### /offers 頁
`/offers` 語氣要像專屬內容頁，不像公開價目表。

適合：

- `快速登入查看`
- `登入後會解鎖這些內容`
- `登入後查看完整費用資訊`
- `登入後查看完整活動場次`

不適合：

- `方案與報價`
- `往下看完整方案`
- `回到方案與報價`

## 尚未串接的部分
目前這次改的是：

- 文案
- 資訊架構
- 視覺上的 locked / unlocked 流程

尚未真正完成的是：

- 真實 LINE Login 驗證
- 登入後狀態切換
- 解鎖後顯示實際內容的邏輯

也就是說，現在頁面已經會正確表達「需要 LINE Login 才能看」，但還沒有真的接上 LINE 身分驗證。

## 建議下一步

1. 決定 LINE Login 的真實串接方式
2. 用登入狀態控制 `/offers` 各區是否解鎖
3. 解鎖後再決定是否保留 blur overlay，或直接切成完整可互動內容
